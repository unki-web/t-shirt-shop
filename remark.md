# remark.md — Замечания и известные ограничения

## 1. Технические ограничения текущей версии

### 1.1 Архитектура
- **Один файл** — весь проект находится в `monochrome-shop.html`. В реальном проекте на Vue 3 код должен быть разбит на компоненты (`.vue` файлы)
- **Нет настоящего роутинга** — навигация реализована через переключение переменной `page`, а не через Vue Router. URL не меняется при переходе между страницами
- **Нет Pinia** — состояние (корзина, товары) хранится в `setup()`. В продакшене следует использовать Pinia store
- **Нет бэкенда** — данные о товарах захардкожены в коде. Нет API, нет базы данных

### 1.2 Данные
- **Нет localStorage** — корзина очищается при перезагрузке страницы
- **Моковые товары** — 8 товаров с SVG-заглушками вместо реальных фотографий
- **Нет реальной оплаты** — форма оплаты имитирует процесс, деньги не списываются
- **Нет отправки email** — подтверждение заказа не отправляется на почту

### 1.3 UX
- **Нет поиска** — отсутствует поиск по названию товара
- **Нет пагинации** — все товары отображаются сразу (при большом каталоге нужен infinite scroll или пагинация)
- **Нет истории заказов** — после оформления заказ нигде не сохраняется
- **Нет личного кабинета** — регистрация и авторизация не реализованы

## 3. Замечания по коду

### 3.1 Что реализовано хорошо
- Использован Vue 3 Composition API — современный подход
- CSS Custom Properties для дизайн-системы — легко менять тему
- Реактивные вычисляемые свойства (`computed`) для корзины и фильтров
- Плавные CSS-переходы на интерактивных элементах

### 3.2 Что улучшить в коде

**Сейчас (монолит):**
```html
<!-- Всё в одном файле — сложно поддерживать -->
<div v-if="page==='home'">...</div>
<div v-if="page==='catalog'">...</div>
<div v-if="page==='product'">...</div>
```

**Правильно (компоненты):**
```
src/
├── views/
│   ├── HomeView.vue
│   ├── CatalogView.vue
│   └── ProductView.vue
├── components/
│   ├── ProductCard.vue
│   ├── CartDrawer.vue
│   └── CheckoutForm.vue
└── stores/
    ├── cart.js
    └── products.js
```

## 5. Выводы

Текущая версия является **учебным прототипом**, демонстрирующим:
- Работу с Vue 3 Composition API
- Построение SPA без сборщика
- Реализацию основного e-commerce flow (каталог → товар → корзина → checkout)
- Минималистичный дизайн с CSS Custom Properties

Для коммерческого использования необходима полная переработка архитектуры с бэкендом, компонентным подходом и реальной платёжной интеграцией.
